5.6 Creating Triggers

A trigger is what causes something to happen with a quest. As soon as a state is activated the following happens:

  • All ‘oninit’ rewards are executed.
  • All triggers are enabled.

At this moment the following triggers from CEL are supported:

  • ‘entersector’: Fires as soon as the camera enters a sector.
  • ‘meshentersector’: Fires as soon as some entity enters a sector.
  • ‘inventory’: Fires as soon as some entity or template enters an inventory.
  • ‘meshselect’: Fires as soon as some entity is selected.
  • ‘message’: Fires as soon as a message is received. You can define your own custom messages but there are also a number of predefined messages. A list of these can be found below.
  • ‘operation’: Not implemented yet.
  • ‘propertychange’: Fires as soon as a property on some entity changes. It is possible to control this trigger to fire based on various conditions.
  • ‘sequencefinish’: Fires as soon as a specific sequence finishes executing.
  • ‘timeout’: Fires as soon as a certain time has elapsed.
  • ‘trigger’: This works in concert with an associated ‘pclogic.trigger’ property class and will fire as soon as the trigger itself fires.
  • ‘watch’: This fires as soon as a certain entity comes close enough to another entity. There are various parameters that you can use to control this trigger.

The ‘message’ trigger understands the following standard messages:

  • ‘ares.Activate’: This message is sent by the Ares Game Controller whenever an object is activated.
